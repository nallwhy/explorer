    View Source Explorer.Query (Explorer v0.4.0-dev)

    High-level query for Explorer.

    Queries convert regular Elixir code which compile to efficient dataframes operations. Variables in queries represent dataframe column names:

    iex> df = Explorer.DataFrame.new(col1: ["a", "b", "c"], col2: [1, 2, 3])
iex> Explorer.DataFrame.filter(df, col2 > 2)
#Explorer.DataFrame<
  Polars[1 x 2]
  col1 string ["c"]
  col2 integer [3]
>

    If you want to pass outside values to the query, you must escape them using ^:

    iex> min = 2
iex> df = Explorer.DataFrame.new(col1: ["a", "b", "c"], col2: [1, 2, 3])
iex> Explorer.DataFrame.filter(df, col2 > ^min)
#Explorer.DataFrame<
  Polars[1 x 2]
  col1 string ["c"]
  col2 integer [3]
>

    All operations from Explorer.Series are imported inside queries. This module also provides operators to use in queries, which are also imported into queries.

    implementation-details

     Implementation details

    Queries simply become lazy dataframe operations at runtime. For example, the following query

    Explorer.DataFrame.filter(df, col2 > 2)

    is equivalent to:

    Explorer.DataFrame.filter_with(df, fn df -> df["col2"] > 2 end)

    This means that, whenever you want to generate queries programatically, you can fallback to the regular _with APIs.
